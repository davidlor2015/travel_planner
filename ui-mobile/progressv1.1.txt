FEATURE 1 — Today Tab: Active Trip Sections + Empty State
================================================================================

Summary:
  Replaced the bare active-trip card (title + "Open live timeline" button only)
  with three lightweight contextual sections. Added a proper empty state for
  users with no trips planned.

Active-trip state — new sections below the existing trip card:

  "Next moment" (amber label)
    - Shows the next upcoming stop pulled from the on-trip snapshot
      (snapshot.next_stop). Displays title + time · location subtitle.
    - Falls back to a soft "You're clear for now. / Open the live timeline
      whenever you're ready." card when no next stop is resolved.

  "A little later" (muted label, conditional)
    - Shows only when a stop exists immediately after next_stop in
      today_stops. Same card layout as "Next moment".
    - Derived by finding next_stop.stop_ref in today_stops[], taking index + 1.

  "Along the way" (muted label, always shown)
    - Soft pressable row: + icon | "Saw something worth noting? Log a stop."
      | chevron. Navigates to the live timeline for unplanned stop logging.

No-active-trip states:

  "Between adventures" state (has upcoming trip)
    - Replaces old "No active trip today." + "Your active trip timeline will
      appear here…" copy.
    - Shows upcoming trip card (title, destination, countdown label) unchanged.
    - Primary CTA changed from "View Trips" to "Plan a trip" → /(tabs)/trips.

  True empty state (no trips at all)
    - Headline: "Nothing on the horizon yet." (Playfair display)
    - Body: "When you plan a trip, Today becomes your quiet travel companion —
      showing what's next, what's later, and what you discover along the way."
    - Primary CTA: "Plan a trip" (espresso pill) → /(tabs)/trips.

Modified files:

  features/today/useTodayModel.ts
    - Added useOnTripSnapshotQuery(activeTrip?.id ?? null) — enabled only when
      an active trip exists; refetchInterval inherited from query definition.
    - Derives nextStop from snapshot.next_stop (null when title is absent).
    - Derives laterStop via useMemo: finds next_stop in today_stops by stop_ref,
      returns the element at index + 1 (or null).
    - TodayModel interface extended: nextStop, laterStop added.
    - Removed TripOnTripSnapshot from imports (only StopSnapshot needed externally).

  features/today/TodayScreen.tsx
    - Full rewrite of both active-trip and no-active-trip branches.
    - Added local SectionLabel and StopCard sub-components to avoid style
      duplication across the three contextual sections.
    - TripOnTripStopSnapshot imported from features/trips/types for StopCard prop.

FEATURE 2 — Profile: Editable Display Name
================================================================================

Summary:
  Users can now set a display name independent of their email address. The
  identity card on the Profile screen becomes a pressable that opens a
  slide-up sheet. Changes are persisted to the backend and reflected
  immediately without a full sign-out/sign-in cycle.

Backend (travel-planner/):

  app/schemas/user.py
    - Added UserUpdate schema: display_name: str | None = None.
    - Pydantic field_validator strips whitespace and enforces 80-char max;
      empty-after-strip becomes None.

  app/api/v1/routes/auth.py
    - Added PATCH /v1/auth/me route.
    - Uses model_fields_set so only explicitly supplied fields are written —
      omitting display_name from the payload is a no-op.
    - Commits, refreshes, returns updated UserResponse.

Mobile (ui-mobile/):

  features/auth/api.ts
    - Added updateMe({ display_name: string }): Promise<MeResponse>
      calling PATCH /v1/auth/me.

  features/auth/hooks.ts
    - Added useUpdateMeMutation() wrapping updateMe.

  providers/AuthProvider.tsx
    - Added refreshUser(): Promise<void> to AuthContextValue and implementation.
    - Calls getMe() and updates local user state in place — no sign-out needed.
    - refreshUser exposed via useAuth().

  shared/api/friendlyError.ts
    - Added "profile" ErrorDomain with 422 and default messages.

  features/profile/EditDisplayNameSheet.tsx  [new file]
    - Slide-up Modal sheet (same pattern as FeedbackSheet, DefaultsSheet).
    - Single TextInput for name, 80-char counter, amber "Profile" eyebrow,
      Playfair "Edit your name" heading.
    - Validation: trimmed length > 0 and ≤ 80 before enabling Save.
    - On save: mutateAsync → refreshUser() → onClose().
    - "Saving…" label while isPending; disabled state on Save button.
    - Error displayed inline using friendlyError(..., "profile").

  app/(tabs)/profile.tsx
    - Imported EditDisplayNameSheet.
    - Added "edit-name" to OpenSheet union type.
    - Identity card converted from static View to Pressable with pencil-outline
      icon (Ionicons, color #C9BCA8) — tapping opens EditDisplayNameSheet.
    - EditDisplayNameSheet rendered at bottom of SafeAreaView alongside
      existing sheets; receives currentName={displayName}.

FEATURE 3 — Budget Tab: Button Hierarchy Restyle
================================================================================

Summary:
  Replaced all loud orange PrimaryButton instances in BudgetTab with two
  small local helpers that match the warm editorial palette. "Add expense" and
  "Save expense" are the clear primary actions; "Save total budget" and
  "Cancel" are visually subordinate.

Design rationale:
  - The shared PrimaryButton uses bg-accent (#B86845, orange) — appropriate
    globally but too saturated in a data-dense screen like Budget.
  - Espresso (#1C1108) is used for primary CTAs throughout the rest of the app
    (profile, auth, today, onTrip) and reads as calmer and more premium.
  - "Save total budget" should not compete visually with the main expense
    workflow — demoted to a quiet bordered secondary.

Modified files:

  features/trips/workspace/BudgetTab.tsx
    - Removed PrimaryButton and SecondaryButton imports (no longer used).
    - Added two local helpers above SUMMARY_TONE_CLASS:

      BudgetPrimaryButton
        bg-espresso text-ivory rounded-2xl min-h-[44px]
        active:opacity-80 | disabled:opacity-40
        Accepts: label, onPress, icon?, disabled?, fullWidth?

      BudgetSecondaryButton
        bg-ivory border border-border text-espresso rounded-2xl min-h-[44px]
        active:opacity-70 | disabled:opacity-40
        Accepts: label, onPress, disabled?, fullWidth?

    - "Save total budget" → BudgetSecondaryButton (quiet, warm ivory, soft border)
    - "Add expense" → BudgetPrimaryButton (espresso, + icon preserved)
    - "Save expense" → BudgetPrimaryButton (espresso)
    - "Cancel" → BudgetSecondaryButton (consistent radius with "Save expense")
    - All handlers, disabled logic, loading text, and form behavior unchanged.

Validation:

  - npx tsc --noEmit (ui-mobile) — PASS (zero errors)
  - No behavior changes to budget save, add expense, or cancel flows.
  - No backend, schema, API, or adapter changes made.
